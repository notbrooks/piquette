# Welcome to Piquette
The Piquette App provides all of the "low-code" tools needed to quickly develop production-quality applications making it the ideal solution for start-ups, dev shops, and creative firms with or without in-house development teams.

This is framework was inspired by [Theo's T3 App Router Tutorial on YouTube](https://www.youtube.com/watch?v=w-l-Z-h-yfI) but everythign was developed internally with a lot of help from Supermaven Pro & some GPT. There's a lot of cool stuff here that Theo has done (insanely well), but I've taken a different approach to the project.  For those who are interested, the name Piquette is a referecne to the Ford Motor Company's factory where the Model T was manufacturing process was developed.  The Model T was the first car to be mass-produced in the United States, and it was a revolutionary car that changed the face of the automotive industry.  The factory was located in Detroit Michigan on Piquette Avenue.  Hence the name.

Piquette is a low-code application meaning that there's a lot that still needs to be developed in order to make it a fully functional application.  My goal here is to make all of the hard stuff simple and let the developer focus on the core functionality of the application.  It's not intened to be a full-featured application, but rather a starting point for anyone who wants to build a full-stack application with Next.js, Postgres, and Vercel.

## Postgres

Postgres is a powerful, open-source object-relational database system.  While you can use any database you want, I've used Postgres because it's a great choice for a low-code application.  It's easy to set up and use, and it's free to use.  Plus, it's a great choice for a start-up because it's easy to scale and it's easy to migrate to a more robust database system in the future.

In addition to Vercel, we are using the Drizzle ORM to interact with Postgres.  Drizzle is a great choice for a low-code application because it's easy to use and it's well-documented.  It's also a great choice for a start-up because it's easy to scale and it's easy to migrate to a more robust ORM in the future.
